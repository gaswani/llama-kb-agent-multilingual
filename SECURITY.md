# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### Please DO NOT:
- Open public GitHub issues for security flaws
- Share exploit details publicly

### Instead:
- Email: info@pathwaystechnologies.com
- Include steps to reproduce and potential impact

## Scope
This project may be deployed in sensitive environments (e.g. government).
Security best practices such as API key protection, HTTPS, and access controls
are strongly recommended for production deployments.
